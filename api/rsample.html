py-rsample: Resampling & Cross-Validation

The py_rsample package provides train/test splitting and cross-validation for time series and general data.

Core Classes

class py_rsample.Split(data, in_id, out_id, id)[source]

Bases: object

Immutable split containing train/test indices.

A Split represents a single train/test division of data, storing: - The original data - Training indices (in_id) - Testing indices (out_id) - Split identifier

data

Original DataFrame

in_id

Training indices (numpy array)

out_id

Testing indices (numpy array)

id

Split identifier string (e.g., Slice001, Fold01)

Example

>>> split = Split(
...     data=df,
...     in_id=np.array([0, 1, 2, 3, 4]),
...     out_id=np.array([5, 6, 7]),
...     id="Slice001"
... )

Main Functions

Train/Test Split

py_rsample.initial_split(data, prop=None, lag=0, date_column=None, train_start=None, train_end=None, test_start=None, test_end=None, **kwargs)

Create initial chronological train/test split for time series.

Supports two modes: 1. Proportion-based (prop/lag): Simple percentage split 2. Explicit dates: Specify exact date ranges for train/test

Parameters:

  • data (DataFrame) DataFrame to split

  • prop (Optional[float]) Proportion for training (e.g., 0.75 for 75% train). If None, uses all but last assessment period

  • lag (Union[str, int, Timedelta]) Gap between train and test (forecast horizon) - int: Number of rows - str: Period like 1 month, 7 days - Timedelta: pandas Timedelta

  • date_column (Optional[str]) Name of date column (required for explicit dates or period lags)

  • train_start (Union[str, datetime, Timestamp, None]) Start date for training data (absolute or relative) - Absolute: datetime, pd.Timestamp, or 2020-01-01 - Relative: start + 1 year, end - 2 years

  • train_end (Union[str, datetime, Timestamp, None]) End date for training data (absolute or relative)

  • test_start (Union[str, datetime, Timestamp, None]) Start date for testing data (absolute or relative)

  • test_end (Union[str, datetime, Timestamp, None]) End date for testing data (absolute or relative)

  • **kwargs Additional arguments (for compatibility)

Return type:

RSplit

Returns:

RSplit object with training/testing methods

Raises:

ValueError If parameters are invalid or date column is missing

Examples

>>> # Mode 1: Proportion-based (backward compatible)
>>> split = initial_time_split(sales_data, prop=0.75)
>>> train = split.training()
>>> test = split.testing()

>>> # With lag (forecast horizon)
>>> split = initial_time_split(
...     sales_data,
...     prop=0.8,
...     lag="1 month",
...     date_column="date"
... )

>>> # Mode 2: Explicit dates - absolute dates
>>> split = initial_time_split(
...     sales_data,
...     date_column="date",
...     train_start="2020-01-01",
...     train_end="2021-12-31",
...     test_start="2022-01-01",
...     test_end="2022-06-30"
... )

>>> # Relative deltas from data boundaries
>>> split = initial_time_split(
...     sales_data,
...     date_column="date",
...     train_start="start",
...     train_end="start + 2 years",
...     test_start="start + 2 years + 1 month",
...     test_end="end"
... )

>>> # Mix of absolute and relative

py_rsample.initial_time_split(data, prop=None, lag=0, date_column=None, train_start=None, train_end=None, test_start=None, test_end=None, **kwargs)[source]

Create initial chronological train/test split for time series.

Supports two modes: 1. Proportion-based (prop/lag): Simple percentage split 2. Explicit dates: Specify exact date ranges for train/test

Parameters:

  • data (DataFrame) DataFrame to split

  • prop (Optional[float]) Proportion for training (e.g., 0.75 for 75% train). If None, uses all but last assessment period

  • lag (Union[str, int, Timedelta]) Gap between train and test (forecast horizon) - int: Number of rows - str: Period like 1 month, 7 days - Timedelta: pandas Timedelta

  • date_column (Optional[str]) Name of date column (required for explicit dates or period lags)

  • train_start (Union[str, datetime, Timestamp, None]) Start date for training data (absolute or relative) - Absolute: datetime, pd.Timestamp, or 2020-01-01 - Relative: start + 1 year, end - 2 years

  • train_end (Union[str, datetime, Timestamp, None]) End date for training data (absolute or relative)

  • test_start (Union[str, datetime, Timestamp, None]) Start date for testing data (absolute or relative)

  • test_end (Union[str, datetime, Timestamp, None]) End date for testing data (absolute or relative)

  • **kwargs Additional arguments (for compatibility)

Return type:

RSplit

Returns:

RSplit object with training/testing methods

Raises:

ValueError If parameters are invalid or date column is missing

>>> # Mode 1: Proportion-based (backward compatible)

>>> train = split.training()
>>> test = split.testing()

Create V-fold cross-validation splits.

Randomly splits data into v groups (folds) for cross-validation. Each fold serves as the test set once while the remaining folds form the training set.

Parameters:

  • data (DataFrame) DataFrame to split

  • v (int) Number of folds (default: 10)

  • repeats (int) Number of times to repeat the v-fold partitioning (default: 1)

  • strata (Optional[str]) Optional column name for stratified sampling

  • seed (Optional[int]) Random seed for reproducibility

Return type:

VFoldCV

Returns:

VFoldCV object containing all splits

Examples

>>> # Basic 5-fold CV
>>> folds = vfold_cv(data, v=5)
>>>
>>> # 10-fold CV with stratification
>>> folds = vfold_cv(data, v=10, strata='outcome')
>>>
>>> # Repeated CV
>>> folds = vfold_cv(data, v=5, repeats=3)
>>>
>>> # Iterate over folds
>>> for fold in folds:
...     train = fold.training()
...     test = fold.testing()
py_rsample.time_series_cv(data, date_column, initial, assess, skip=0, cumulative=True, lag=0, slice_limit=None)[source]

Create time series cross-validation splits.

Creates multiple chronological train/test splits for time series validation. Supports both rolling window (fixed training size) and expanding window (growing training size).

Parameters:

  • data (DataFrame) DataFrame with time series data

  • date_column (str) Name of date column

  • initial (Union[str, int, Timedelta]) Initial training period size - int: Number of rows - str: Period like 2 years, 6 months - Timedelta: pandas Timedelta

  • assess (Union[str, int, Timedelta]) Assessment (test) period size

  • skip (Union[str, int, Timedelta]) Gap between consecutive folds (default 0)

  • cumulative (bool) If True, use expanding window; if False, use rolling window

  • lag (Union[str, int, Timedelta]) Gap between train and test (forecast horizon)

  • slice_limit (Optional[int]) Maximum number of folds to create (default None = all folds)

Return type:

TimeSeriesCV

Returns:

TimeSeriesCV object (iterable of RSplit objects)

Examples

>>> # Expanding window: 2-year initial, 3-month assessment
>>> cv = time_series_cv(
...     data=sales_data,
...     date_column="date",
...     initial="2 years",
...     assess="3 months",
...     cumulative=True
... )
>>> for split in cv:
...     train = split.training()
...     test = split.testing()
